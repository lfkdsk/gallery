# Animal Index Guide — 动物索引操作指南

## 概述

`animal_index.json` 收录了相册中所有含真实动物的图片，按动物种类（中英双语）归类，值为 `文件夹/文件名.扩展名` 格式的路径列表。

本文档说明如何在未来的会话中继续扫描尚未处理的文件夹，以及如何向索引添加结果。

## 扫描方法

### 工具
- 在相册仓库根目录设置路径变量：
  - `GALLERY_ROOT=gallery`
  - `THUMB_ROOT=<thumbnail-worktree>/gallery`
  - `WORK_DIR=<scratch-output-dir>`
- 使用缩略图分支：`git worktree add <thumbnail-worktree> thumbnail`
- 缩略图为 1000px WebP，避免 Claude 多图请求 2000px 维度限制
- 若缩略图不存在，fallback 到原图（注意原图可能更大）

### 推荐流程：缩略图联系表 / sprite sheet

比起逐张读取图片，更推荐先把一个文件夹生成一张带编号和文件名的联系表（sprite/contact sheet），一次性肉眼筛出疑似动物图，再只放大少数需要确认的图片。这样更省 token，也更容易看出同一动物的连续照片。

示例脚本（在 Codex 工作目录中生成联系表，不改相册原文件）：

```bash
python3 - <<'PY'
from PIL import Image, ImageDraw, ImageFont
import os, math

base = os.environ.get('THUMB_ROOT', '<thumbnail-worktree>/gallery')
out = os.path.join(os.environ.get('WORK_DIR', '.'), 'contact_sheets')
folders = ['BayAnim']
os.makedirs(out, exist_ok=True)

try:
    font = ImageFont.truetype(os.environ['FONT_PATH'], 15)
except Exception:
    font = ImageFont.load_default()

for folder in folders:
    dirp = os.path.join(base, folder)
    files = sorted(f for f in os.listdir(dirp) if f.lower().endswith(('.webp', '.jpg', '.jpeg', '.png')))
    tw, th, lh, cols = 180, 135, 40, 5
    rows = math.ceil(len(files) / cols) or 1
    sheet = Image.new('RGB', (cols * tw, rows * (th + lh)), 'white')
    draw = ImageDraw.Draw(sheet)

    for i, name in enumerate(files):
        im = Image.open(os.path.join(dirp, name)).convert('RGB')
        im.thumbnail((tw, th), Image.Resampling.LANCZOS)
        x = (i % cols) * tw + (tw - im.width) // 2
        y = (i // cols) * (th + lh)
        sheet.paste(im, (x, y))
        draw.text(((i % cols) * tw + 4, y + th + 2), f'{i + 1}. {name}'[:35], fill=(0, 0, 0), font=font)

    sheet.save(os.path.join(out, folder + '_contact.jpg'), quality=90)
PY
```

判读建议：
- 第一轮只看联系表，记录“确定有真实动物”的编号和文件名；雕塑、卡通、壁画、脊兽、器物纹饰不收录。
- 对太小、逆光、相近物种（如 Western Grebe / Clark's Grebe、鹰类、海鸟）再单独打开缩略图原图确认。
- 连续照片应逐张记录路径；同一张照片里有多个真实动物种类时，可以在中间 JSON 中把 `zh`/`en` 写成多个物种，合并时分别进入对应 key。
- 如果缩略图存在但原图目录找不到同 basename 的文件，视为孤儿缩略图，跳过，不要写入 `animal_index.json`。
- 物种不确定时宁可用较保守的 key（例如 `Gull sp.`），不要过度精确。

### 文件扩展名对照
- 缩略图分支将所有图片统一转为 `.webp`
- 原图可能为 `.jpg`、`.jpeg`、`.webp` 等
- 写入 `animal_index.json` 时须使用原图的实际扩展名，例如：
  - 缩略图 `Daily/final-4.webp` → 原图 `Daily/final-4.jpg`
  - 核查方式：`ls "$GALLERY_ROOT/<folder>/<basename>".*`

### Agent 扫描模板

启动 background agent 扫描一个文件夹（以 `BayAnim/` 为例）：

```
1. 读取 `$THUMB_ROOT/BayAnim/` 中所有图片
2. 对每张图片使用 Read 工具查看
3. 如果图片中有真实动物（非雕塑/卡通/壁画），记录：
   - 文件路径（原图格式）：BayAnim/filename.ext
   - 动物种类（中英双语，尽量精确到亚种）
4. 把结果写入 `$GALLERY_ROOT/.animal_scan_results/<folder>.json`
   格式：{ "folder/file.ext": {"zh": "...", "en": "..."} }
5. 每处理完 5-10 张就写一次文件（增量写，防止 context 积累导致失败）
```

### 合并结果到主索引

扫描完成后，将 `.animal_scan_results/<folder>.json` 中的结果手动或用脚本合并到 `animal_index.json`：
- 按动物种类分组
- 去重（同一图片不重复收录）
- 用相同的中英双语 key 格式
- 合并时用原图目录按 basename 查实际扩展名；找不到原图的缩略图不要合并
- 合并后更新本文档中的扫描状态和动物结果摘要

### 合并后校验

```bash
python3 -m json.tool "$GALLERY_ROOT/animal_index.json" >/dev/null
python3 -m json.tool "$GALLERY_ROOT/.animal_scan_results/<folder>.json" >/dev/null
```

检查新增扫描结果是否有缺失路径：

```bash
python3 - <<'PY'
import json, os
base = os.environ.get('GALLERY_ROOT', 'gallery')
scan = json.load(open(os.path.join(base, '.animal_scan_results/<folder>.json'), encoding='utf-8'))
missing = [p for p in scan if not os.path.exists(os.path.join(base, p))]
print('missing', len(missing))
for p in missing:
    print(p)
PY
```

## 动物种类格式规范

Key 格式：`中文种名·颜色变体 / English Species (Scientific Name)`

例：
- `中华田园猫·三花 / Chinese Domestic Cat (Calico / Tricolor)`
- `东部灰松鼠 / Eastern Gray Squirrel (Sciurus carolinensis)`
- `阿拉斯加海岸棕熊 / Alaska Coastal Brown Bear (Ursus arctos middendorffi)`

猫咪品种由于数量多，拆分为多个颜色变体 key。其他动物尽量精确到亚种。
